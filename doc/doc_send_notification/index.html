Úvod

Adresa API

K posílání Push notifikací se využívá REST API. Pomocí HTTP dotazů komunikujeme s Firebase serverem.

Push notifikace se posílá dotazem POST na adresu Firebase serveru. Vycházím z dokumentace Firebase

                                    POST 'https://fcm.googleapis.com/fcm/send'

Otestování notifikace

Můžete otestovat pomocí příkazového řádku.

                                  curl -X POST 'https://fcm.googleapis.com/fcm/send' \
                                  --header "Authorization: key=AQH0uW0FvjaoiF..."\
                                  --Header "Content-Type: application/json" \
                                  -d "{
                                      "priority": "High",
                                      "to": "f88Kf...",
                                      "notification":{
                                         "title": "Bzzzzzzz",
                                         "body": "Uletely vcely",
                                         "badge": "1",
                                         "sound": "default"
                                       },
                                      }"

Hlavička 

                                  --header "Authorization: key=AQH0uW0FvjaoiF..."\
                                  --Header "Content-Type: application/json" \

Autorizace

Authorization string povinný

Uvádí se ve tvaru key=server_key.

server_key je vygenerovaný pro aplikaci ve Firebase.

Pro ForsageApp je to:

                                  AAAA2chMxxs:APA91bG4zUPCLPGF5HI6OfzTg0PjSIl-zwu_EmRt3Zd1wEJirX5XB3Rb5gpB6kqCQ_xhFc6C-qQl0Ag00X6d-isRO0g7s1Dby9lzeqGxhyKjbpCIlKA81XZlS9wokOuQH0uW0FvjaoiF

Content-Type

Content-Typestring povinný

Uvádíme application/json.

Tělo dotazu

Tělo dotazu je ve formátu JSON.

                                  {
                                    "to":"f88Kf9X6...",
                                    "priority":"high",
                                    "notification":{
                                        ...
                                      },
                                  }

Cíl dotazu

to string volitelný

Identifikuje zařízení, dosazujeme Push Notification Token

registration_ids array of strings volitelný

Identifikuje vícero zařízení na které chceme notifikaci poslat. Dosazujeme pole Push Notification Tokenů

Možnosti

priority string volitelný

Nastaví prioritu zprávy. Ve výchozím nastavení jsou oznámení zasílána s vysokou prioritou a datové zprávy s normální prioritou. Normální priorita optimalizuje spotřebu baterie klientské aplikace a měla by být použita, pokud není vyžadováno okamžité doručení. U zpráv s normální prioritou může aplikace přijmout zprávu s neurčeným zpožděním. Když je zpráva odeslána s vysokou prioritou, je odeslána okamžitě a aplikace může zobrazit oznámení.

Možné hodnoty jsou: high, normal

time_to_live number volitelný

Specifikuje, jak dlouho (v sekundách) notifikace bude na zařízení. Defaultně je nastaveno 4 týdny.

notification object volitelný

Specifikuje notifikaci. Více níže.

data object volitelný

Můžeme posílat data. Na příklad: {"score":"3x1"}:

Více v dokumentaci Firebase.

Objekt notification

                                  "notification":{
                                       "title": "Bzzzzzzz",
                                       "body": "Uletely vcely",
                                       "badge": "1",
                                       "sound": "default",
                                       "color":"#ffc107",
                                     }

Společné

title string volitelný

Titulek notifikace. V dokumentaci se uvadí, že není vidět na iOS mobilnech a tabletech. Netestoval jsem to.

body string volitelný

Tělo notifikace.

sound string volitelný

Zvuk, který se přehraje, když příjde notifikace. string specifikuje zvuk v klientské aplikaci.

Pro iOS je uložen v Library/Sounds.

Pro Android je uložen v /res/raw/.

Při hodnotě: default se přehraje defaultní zvuk.

click_ation string volitelný

Akce, která se vykoná, když uživatel klikne na notifikaci.

U iOS odpovídá category v APN payload.

U Android odpovídá navstavenému filtru.

Android

android_channel_id string volitelný

Před přijetím jakéhokoli oznámení s tímto ID kanálu musí aplikace vytvořit kanál s tímto ID kanálu.

Pokud není zadána, vezme se z app manifestu.

iconstring volitelný

Ikona notifikace

Nastaví ikonu z myicon. Pokud není nastavená, vezme se z app manifestu.

colorstring volitelný

Barva notifikace. V #rrggbb formátu.

iOS

subtitle string volitelný

Podtitulek notifikace

badge string volitelný

Hodnota počtu oznámení na ikoně.

Když není specifikováno, nemění se. Pokud je 0, počet oznámení se odstraní.

Více v dokumentaci Firebase.

Odpověď serveru na požadavek

                                  {
                                    "multicast_id":6505292741566089650,
                                    "success":0,
                                    "failure":1,
                                    "canonical_ids":0,
                                    "results":[{
                                      "error":"NotRegistered"
                                    }]
                                  }

multicast_id string

ID identifikující multicastovou zprávu.

success string

Počet odeslaných zpráv bez erroru.

failure string

Počet neodeslaných zpráv

results Array of objects

Pole objektů se statusem zprávy

Objekt result

message_id number

ID zprávy, která se úspěšně odeslala.

error string

Nejčastější errory

  • NotRegistered - Neregistrované FCM
  • MissingRegistration - Neobsahuje cíl(to), kam se má poslat.
  • Authenticate Error - Špatné ID serveru
  • InvalidRegistration - Špatný formát FCM

    • Více o možnostech erroru v dokumentaci Firebase.
